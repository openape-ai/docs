# OpenApe Grants

## @openape/grants

The permission engine. Framework-agnostic. This package provides the core grant lifecycle, AuthZ-JWT issuance, and verification — independent of any framework.

### Grant Lifecycle

```
Request → Pending → Approved/Denied → (if approved) Active → Used/Expired/Revoked
```

### Grant Types

| Type | Behavior | AuthZ-JWT Lifetime | Reusable? |
|---|---|---|---|
| `once` | Single use — consumed after first use | 5 minutes | No |
| `timed` | Valid for a time window (TTL) | Until `expires_at` | Yes |
| `always` | Standing permission — active until revoked | 1 hour (renewable) | Yes |

For `timed` grants, specify a `duration` in seconds when requesting. For `always` grants, the AuthZ-JWT expires after 1 hour but can be re-fetched as long as the grant is not revoked.

### AuthZ-JWT

On approval, a signed AuthZ-JWT is issued:

```json
{
  "iss": "https://id.example.com",
  "sub": "agent@example.com",
  "aud": "target-system",
  "target_host": "prod-server.example.com",
  "grant_id": "uuid",
  "grant_type": "once",
  "permissions": ["deploy"],
  "command": ["systemctl", "restart", "nginx"],
  "cmd_hash": "sha256:a1b2c3...",
  "decided_by": "alice@example.com",
  "exp": 1234567890,
  "jti": "unique-grant-id"
}
```

Key security features:
- **`aud` binding** — token only valid for the intended target service
- **`target_host`** — restricts to a specific host
- **`cmd_hash`** — binds to exact command (prevents substitution attacks)
- **`decided_by`** — dual accountability (agent identity ≠ approver identity)
- **`jti`** — replay protection
- **Expiry** — all grants have a maximum lifetime

### Grant Request Fields

| Field | Required | Description |
|---|---|---|
| `requester` | Yes | Agent email (auto-set if using agent token) |
| `target_host` | Yes | Host where the grant is valid |
| `audience` | Yes | Service identifier (e.g., `escapes`, `proxy`) |
| `grant_type` | No | `once` (default), `timed`, or `always` |
| `command` | No | Command array (e.g., `["apt-get", "upgrade"]`) |
| `permissions` | No | Permission strings |
| `duration` | For `timed` | Duration in seconds |
| `reason` | No | Human-readable reason for the request |
| `run_as` | No | Execute as this user |

## Grant Routes in `@openape/nuxt-auth-idp`

Grant management is integrated into `@openape/nuxt-auth-idp`. Enable grant pages with `grants.enablePages: true` in your module config.

::callout{type="warning"}
`@openape/nuxt-grants` is **deprecated**. All grant functionality has been consolidated into `@openape/nuxt-auth-idp`.
::

**Auto-registered API routes:**
- `POST /api/grants` — create grant request
- `GET /api/grants` — list grants
- `GET /api/grants/:id` — get grant details (supports ETag polling)
- `POST /api/grants/:id/approve` — approve a grant (returns AuthZ-JWT)
- `POST /api/grants/:id/deny` — deny a grant
- `POST /api/grants/:id/revoke` — revoke an active grant
- `POST /api/grants/:id/token` — get AuthZ-JWT for approved grant
- `POST /api/grants/:id/consume` — consume a `once` grant
- `POST /api/grants/verify` — verify an AuthZ-JWT

**Agent API routes:**
- `POST /api/agent/enroll` — register a new agent (requires Management Token)
- `POST /api/agent/challenge` — request auth challenge
- `POST /api/agent/authenticate` — authenticate with signed challenge

**Pages** (overridable, enabled via `grants.enablePages: true`):
- `/grants` — grant dashboard
- `/grant-approval` — approve/deny UI
- `/enroll` — agent enrollment form

For detailed request/response schemas, see the [Agent Integration Guide](/getting-started/developers).

## Agent Tools

The following tools use the grant system for access control:

| Tool | Purpose | Docs |
|---|---|---|
| [grapes](/ecosystem/grapes) | Universal Grant Management CLI | Request, approve, delegate, execute |
| [escapes](/ecosystem/escapes) | Local privilege elevation via AuthZ-JWT | Setuid-root Rust binary |
| [@openape/proxy](/ecosystem/proxy) | Agent HTTP gateway with grant-based access rules | Forward proxy |
| [@openape/browser](/ecosystem/browser) | Grant-aware headless browser | Playwright wrapper |
| [@openape/shapes](/ecosystem/shapes) | Grant-aware CLI wrappers | Adapters for kubectl, aws, etc. |
