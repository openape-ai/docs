# OpenApe Grants

## @openape/grants

The permission engine. Framework-agnostic.

### Grant Lifecycle

```
Request → Pending → Approved/Denied → (if approved) Active → Used/Expired/Revoked
```

### Grant Types

| Type | Behavior |
|---|---|
| `once` | Single use — consumed after first use |
| `timed` | Valid for a time window (TTL) |
| `always` | Standing permission — active until revoked |

### AuthZ-JWT

On approval, a signed AuthZ-JWT is issued:

```json
{
  "sub": "agent@example.com",
  "aud": "target-system",
  "grant_type": "once",
  "permissions": ["deploy"],
  "cmd_hash": "sha256:a1b2c3...",
  "decided_by": "alice@example.com",
  "exp": 1234567890,
  "jti": "unique-grant-id"
}
```

Key security features:
- **`aud` binding** — token only valid for the intended target
- **`cmd_hash`** — binds to exact command (prevents substitution attacks)
- **`decided_by`** — dual accountability (agent owner ≠ approver)
- **`jti`** — replay protection
- **Expiry** — all grants have a maximum lifetime

## Grant Routes in `@openape/nuxt-auth-idp`

Grant management is integrated into `@openape/nuxt-auth-idp`. Enable grant pages with `grants.enablePages: true` in your module config.

::callout{type="warning"}
`@openape/nuxt-grants` is **deprecated**. All grant functionality has been consolidated into `@openape/nuxt-auth-idp`.
::

**Auto-registered routes (via nuxt-auth-idp):**
- `/api/grants` — list and create grant requests
- `/api/grants/:id` — get grant details
- `/api/grants/:id/approve` — approve a grant
- `/api/grants/:id/deny` — deny a grant
- `/api/grants/:id/revoke` — revoke an active grant
- `/api/grants/:id/token` — issue AuthZ-JWT for approved grant
- `/api/grants/verify` — verify an AuthZ-JWT
- `/api/agent/enroll` — register a new agent
- `/api/agent/challenge` — request auth challenge
- `/api/agent/authenticate` — authenticate with signed challenge

**Pages** (overridable, enabled via `grants.enablePages: true`):
- `/grants` — grant dashboard
- `/grant-approval` — approve/deny UI
- `/enroll` — agent enrollment form

## escapes

A setuid-root Rust binary for local privilege elevation via the grant system. Receives a pre-approved AuthZ-JWT and executes the authorized command.

```bash
# Install
cargo build --release
sudo make install  # installs to /usr/local/bin/escapes with setuid

# Execute with grant JWT (obtained via grapes)
escapes --grant <jwt> -- apt-get upgrade
escapes --grant-stdin -- systemctl restart nginx
escapes --grant-file /path/to/jwt -- apt-get upgrade
escapes --run-as deploy --grant <jwt> -- systemctl restart nginx
```

### Security Model

- **Setuid binary** — starts as root
- **7-step JWT verification** — issuer, signature, approver, audience, `target_host`, `cmd_hash`, IdP consume check
- **Environment sanitized** — `LD_PRELOAD`, `PATH` etc. reset before exec
- **Audit log** — every execution logged as JSONL to `/var/log/openape/audit.log`
- **Config** — root-owned `/etc/openape/config.toml`
- **Exit codes** — `0` (success), `1` (config error), `5` (JWT failed), `126` (exec failed), `127` (not found)

## grapes

Universal Grant Management CLI. Handles authentication, grant requests, approvals, delegations, and integrates with `escapes` for privilege elevation.

```bash
# Login
grapes login --idp https://id.openape.at --key ~/.ssh/id_ed25519

# Request a grant
grapes request --target "api.github.com" --permissions "write:issues"
grapes request-cmd "apt-get upgrade" --target prod-server --approval once

# Request and execute in one step (uses escapes)
grapes run escapes "systemctl restart nginx" --approval once

# Approve/deny (as approver)
grapes approve <grant-id>
grapes deny <grant-id>

# Delegations
grapes delegate --to agent@example.com --with "read:repos"
grapes delegations
```

### Commands

| Command | Description |
|---|---|
| `login` | Authenticate to IdP |
| `logout` | Clear local credentials |
| `whoami` | Show current identity |
| `request` | Request a grant |
| `request-cmd` | Request grant for a specific command |
| `request-capability` | Request capability-based grant |
| `list` | List active grants |
| `status` | Show grant status |
| `token` | Get AuthZ-JWT for approved grant |
| `revoke` | Revoke a grant |
| `approve` | Approve pending grant |
| `deny` | Deny pending grant |
| `run` | Request grant and execute command via `escapes` |
| `delegate` | Create delegation to another identity |
| `delegations` | List active delegations |

## @openape/proxy

Agent HTTP gateway — a forward proxy that sits between an agent and the internet, enforcing grant-based access rules before forwarding requests.

```bash
# Start the proxy
openape-proxy --config config.toml
openape-proxy --config config.toml --dry-run
openape-proxy --config config.toml --mandatory-auth
```

### Configuration (TOML)

```toml
listen = "127.0.0.1:9090"
idp_url = "https://id.example.com"
agent_email = "bot@example.com"
default_action = "block"  # block | request | request-async
audit_log = "/var/log/openape-proxy/audit.jsonl"

[[rules.allow]]
domains = ["api.github.com"]
methods = ["GET"]

[[rules.grant_required]]
domains = ["api.github.com"]
methods = ["POST", "PUT", "DELETE"]
grant_type = "once"

[[rules.deny]]
domains = ["*.internal.corp"]
```

### Features

- **Rule evaluation** — `allow`, `deny`, `grant_required` per domain/path/method
- **Multi-agent** — support multiple agent identities
- **Audit logging** — all requests logged as JSONL
- **Dry-run mode** — evaluate rules without enforcing

## @openape/browser

Grant-aware headless browser — a Playwright wrapper with route interception, grant enforcement, and delegation login.

```typescript
import { createGrantedBrowser } from '@openape/browser'

const browser = await createGrantedBrowser({
  rulesFile: './rules.toml',
  idpUrl: 'https://id.example.com',
  agentEmail: 'bot@example.com'
})

const page = await browser.newPage()
await page.goto('https://target-app.example.com')

// Login as delegated user
await browser.loginAs({
  email: 'alice@example.com',
  delegationToken: '<jwt>'
})

await browser.close()
```

### Features

- **Route interception** — all HTTP(S) requests evaluated against rules (allow/deny/grant_required)
- **Delegation login** — login as another user via delegation tokens
- **Rule formats** — TOML-based rules, same format as `@openape/proxy`
- **Playwright-native** — full access to underlying Playwright `BrowserContext`

## @openape/shapes

Grant-aware CLI wrappers — adapters that make existing CLIs (kubectl, aws, gcloud, etc.) grant-aware without modifying their source.

```bash
# Search available adapters
shapes adapter search kubectl

# Install an adapter
shapes adapter install kubectl

# Explain what permissions a command needs
shapes explain "kubectl delete pod nginx -n production"

# Execute with grant interception
shapes execute "kubectl delete pod nginx -n production"

# Request capability-based grant
shapes request --capability "kubernetes:delete:pod"
```

### Concepts

- **Adapters** — plugins that wrap CLI tools with grant awareness
- **Operations** — define resources/actions each adapter can perform
- **Resource Chains** — hierarchical resource selectors (e.g., `repository:owner,name` → `issue:id`)
- **Capabilities** — high-level permissions mapped to CLI operations
- **Registry** — central registry for discovering and installing adapters
