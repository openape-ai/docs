# Ecosystem

OpenApe isn't a monolith — it's a set of small, focused packages you compose as needed. Use one. Use all. Each works standalone.

## Architecture

```
┌──────────────────────────────────────────────────┐
│              Framework Modules                   │
│   nuxt-auth-idp   nuxt-auth-sp   nuxt-grants    │
├──────────────────────────────────────────────────┤
│              Protocol Packages                   │
│          @openape/auth     @openape/grants       │
├──────────────────────────────────────────────────┤
│              Foundation                          │
│              @openape/core                       │
├──────────────────────────────────────────────────┤
│       Agent Tools & Infrastructure               │
│  @openape/proxy  @openape/browser  @openape/shapes│
│  grapes (CLI)    escapes (Rust)                  │
└──────────────────────────────────────────────────┘
```

## Packages

| Package | Description | Layer |
|---|---|---|
| `@openape/core` | DNS discovery, crypto, PKCE, JWT utilities | Foundation |
| [`@openape/auth`](/ecosystem/auth) | OIDC login protocol — IdP and SP sides | Protocol |
| [`@openape/grants`](/ecosystem/grants) | Grant lifecycle, AuthZ-JWT issuance | Protocol |
| `@openape/nuxt-auth-idp` | Drop-in Nuxt module: run your own IdP | Framework |
| [`@openape/nuxt-auth-sp`](/ecosystem/nuxt-auth-sp) | Drop-in Nuxt module: login via OpenApe | Framework |
| [`@openape/proxy`](/ecosystem/proxy) | Agent HTTP gateway — forward proxy with grant-based access control | Agent Tools |
| [`@openape/browser`](/ecosystem/browser) | Grant-aware headless browser — Playwright wrapper with route interception | Agent Tools |
| [`@openape/shapes`](/ecosystem/shapes) | Grant-aware CLI wrappers — adapters for existing CLIs (kubectl, aws, etc.) | Agent Tools |
| [`grapes`](/ecosystem/grapes) | Universal Grant Management CLI — request, approve, delegate, execute | Agent Tools |
| [`escapes`](/ecosystem/escapes) | Setuid-root Rust binary for local privilege elevation via AuthZ-JWT | OS-level |

## Combinations

| Use Case | Packages |
|---|---|
| App with OpenApe login | `nuxt-auth-sp` |
| Run your own IdP | `nuxt-auth-idp` |
| Full IdP + Grants | `nuxt-auth-idp` (with `grants.enablePages: true`) |
| SP with grant requests | `nuxt-auth-sp` + `@openape/grants` |
| Non-Nuxt integration | `@openape/auth` + `@openape/grants` |
| Agent CLI with grants | `@openape/grapes` |
| Local privilege elevation | `escapes` + `@openape/grapes` |
| Agent HTTP proxy | `@openape/proxy` |
| Agent browser automation | `@openape/browser` |
| Grant-aware CLI tools | `@openape/shapes` + `@openape/grapes` |

## Design Principles

1. **Separation** — Auth ≠ Grants. Not every app needs both.
2. **Layered** — Core → Protocol packages → Framework modules → Apps
3. **Stateless SP** — Service providers need zero server storage.
4. **Framework-agnostic core** — Nuxt modules are convenience, not requirement.
5. **Passkeys-only** — No passwords. NIS2 compliant by design.
6. **Minimal tokens** — AuthN says *who*, AuthZ says *what may they do*.
