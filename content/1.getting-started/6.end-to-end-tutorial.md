# End-to-End Tutorial

This tutorial walks through setting up a complete OpenApe environment on your local machine: an Identity Provider (IdP), a Service Provider (SP) app, and an AI agent that requests grants.

## What We'll Build

```
┌──────────────────┐     DDISA      ┌──────────────────┐
│   Service Provider│◄──────────────►│  Identity Provider│
│   (Nuxt App)      │   Login Flow   │  (Nuxt + IdP)    │
│   localhost:3001   │               │  localhost:3000    │
└──────────────────┘               └──────────────────┘
                                            ▲
                                            │ Grant Request
                                            │ + Approval
                                   ┌────────┴─────────┐
                                   │  AI Agent (CLI)   │
                                   │  grapes + escapes │
                                   └──────────────────┘
```

By the end, you'll have:
1. An IdP that issues Passkey-based logins and manages grants
2. An SP app that authenticates users via DDISA
3. An enrolled agent that can request and use grants

## Part 1: Set Up the IdP

### Create the IdP project

```bash
npx nuxi@latest init my-idp
cd my-idp
npm install @openape/nuxt-auth-idp
```

### Configure the IdP

Add the module to `nuxt.config.ts`:

```typescript
export default defineNuxtConfig({
  modules: ['@openape/nuxt-auth-idp'],
  openapeIdp: {
    rpName: 'My OpenApe IdP',
    rpID: 'localhost',
    rpOrigin: 'http://localhost:3000',
    managementToken: 'dev-management-token-change-in-prod',
    grants: {
      enablePages: true
    }
  }
})
```

### Start the IdP

```bash
npm run dev -- --port 3000
```

Verify it's running:

```bash
# JWKS endpoint
curl http://localhost:3000/.well-known/jwks.json

# OIDC discovery
curl http://localhost:3000/.well-known/openid-configuration
```

## Part 2: Set Up the SP

In a separate terminal:

### Create the SP project

```bash
npx nuxi@latest init my-sp
cd my-sp
npm install @openape/nuxt-auth-sp
```

### Configure the SP

```typescript
export default defineNuxtConfig({
  modules: ['@openape/nuxt-auth-sp'],
  openapeSp: {
    fallbackIdpUrl: 'http://localhost:3000'
  }
})
```

### Add a Login Page

Create `app/pages/index.vue`:

```vue
<template>
  <div style="max-width: 400px; margin: 50px auto;">
    <h1>My App</h1>
    <div v-if="user">
      <p>Logged in as: <strong>{{ user.email }}</strong></p>
      <p>Type: {{ user.act }}</p>
      <button @click="logout()">Logout</button>
    </div>
    <OpenApeAuth v-else />
  </div>
</template>

<script setup>
const { user, logout } = useOpenApeAuth()
</script>
```

### Start the SP

```bash
npm run dev -- --port 3001
```

## Part 3: Register a Human User

### Create a Registration URL

```bash
curl -X POST http://localhost:3000/api/admin/registration-urls \
  -H "Authorization: Bearer dev-management-token-change-in-prod" \
  -H "Content-Type: application/json" \
  -d '{"email": "alice@localhost", "name": "Alice"}'
```

This returns a one-time registration URL. Open it in your browser to register a Passkey.

::callout{type="info"}
In development on `localhost`, Passkeys use the device's built-in authenticator. On macOS, this uses Touch ID or iCloud Keychain.
::

## Part 4: Human Logs In to the SP

1. Open `http://localhost:3001` in your browser
2. Enter `alice@localhost` in the login form
3. Since there's no DNS record for `localhost`, the SP uses the fallback IdP (`http://localhost:3000`)
4. Authenticate with your Passkey
5. You're redirected back to the SP, logged in as `alice@localhost`

This is the complete DDISA flow: email → IdP discovery → Passkey auth → JWT → SP session.

## Part 5: Enroll an Agent

### Generate the agent's key

```bash
ssh-keygen -t ed25519 -f /tmp/agent_key -N ""
```

### Register the agent with the IdP

```bash
PUBLIC_KEY=$(cat /tmp/agent_key.pub)

curl -X POST http://localhost:3000/api/agent/enroll \
  -H "Authorization: Bearer dev-management-token-change-in-prod" \
  -H "Content-Type: application/json" \
  -d "{
    \"email\": \"agent+worker@localhost\",
    \"name\": \"test-worker\",
    \"publicKey\": \"$PUBLIC_KEY\"
  }"
```

### Authenticate the agent

```bash
# Get a challenge
CHALLENGE=$(curl -s -X POST http://localhost:3000/api/agent/challenge \
  -H "Content-Type: application/json" \
  -d '{"agent_id": "agent+worker@localhost"}' | jq -r '.challenge')

echo "Challenge: $CHALLENGE"

# Sign the challenge
SIGNATURE=$(echo -n "$CHALLENGE" | openssl pkeyutl -sign -inkey /tmp/agent_key -rawin | base64)

# Authenticate
RESPONSE=$(curl -s -X POST http://localhost:3000/api/agent/authenticate \
  -H "Content-Type: application/json" \
  -d "{
    \"agent_id\": \"agent+worker@localhost\",
    \"challenge\": \"$CHALLENGE\",
    \"signature\": \"$SIGNATURE\"
  }")

AGENT_TOKEN=$(echo "$RESPONSE" | jq -r '.token')
echo "Agent token: $AGENT_TOKEN"
```

## Part 6: Agent Requests a Grant

The agent requests permission to perform a privileged action:

```bash
GRANT_RESPONSE=$(curl -s -X POST http://localhost:3000/api/grants \
  -H "Authorization: Bearer $AGENT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "requester": "agent+worker@localhost",
    "target_host": "localhost",
    "audience": "escapes",
    "grant_type": "once",
    "command": ["echo", "Hello from OpenApe!"],
    "reason": "Testing the grant flow"
  }')

GRANT_ID=$(echo "$GRANT_RESPONSE" | jq -r '.id')
echo "Grant ID: $GRANT_ID"
echo "Status: $(echo "$GRANT_RESPONSE" | jq -r '.status')"
```

The grant is now `pending`. Check the IdP's grant dashboard at `http://localhost:3000/grants` — you'll see the pending request.

## Part 7: Approve and Use the Grant

### Approve via API

```bash
APPROVE_RESPONSE=$(curl -s -X POST "http://localhost:3000/api/grants/$GRANT_ID/approve" \
  -H "Authorization: Bearer dev-management-token-change-in-prod")

AUTHZ_JWT=$(echo "$APPROVE_RESPONSE" | jq -r '.authz_jwt')
echo "AuthZ-JWT received!"
```

### Verify the token

```bash
curl -s -X POST http://localhost:3000/api/grants/verify \
  -H "Content-Type: application/json" \
  -d "{\"token\": \"$AUTHZ_JWT\"}" | jq
```

You'll see the full claims including `sub`, `aud`, `cmd_hash`, `decided_by`, and `grant_type`.

### Consume the grant (for `once` grants)

```bash
curl -s -X POST "http://localhost:3000/api/grants/$GRANT_ID/consume" \
  -H "Authorization: Bearer $AUTHZ_JWT" | jq
```

Response: `{ "status": "consumed" }` — the grant is now used and cannot be reused.

## Part 8: Using grapes CLI

The `grapes` CLI simplifies the entire agent flow into a few commands:

```bash
# Install grapes
npm install -g @openape/grapes

# Login as agent
grapes login --idp http://localhost:3000 --key /tmp/agent_key --email agent+worker@localhost

# Check identity
grapes whoami

# Request a grant and wait for approval
grapes request "echo Hello" --audience escapes --wait &

# In another terminal, approve it
grapes login --idp http://localhost:3000  # Login as human
grapes list --status pending
grapes approve <grant-id>
```
